# Custom Agent 運用ガイド

このドキュメントは、オーケストレーター（GitHub Copilot）とサブエージェントを使った継続開発フローを定義します。

## 役割

- オーケストレーター（Copilot）
  - タスク提案、優先度管理、実施順制御、最終反映（テスト/コミット/push）を担当。
- サブエージェントA（調査）
  - 影響範囲の調査、対象ファイル特定、設計差分案作成を担当。
- サブエージェントB（実装）
  - root/template 同期でのコード変更を担当。
- サブエージェントC（検証）
  - 対象テスト、全体テスト、実行スモークを担当。

## 標準フロー（順次実行）

1. オーケストレーターが候補タスクを提示する。
2. ユーザーが実施対象を選択する。
3. サブエージェントAで調査結果を取得する。
4. サブエージェントBで実装する（root/template 同期）。
5. サブエージェントCで検証する（対象→全体）。
6. オーケストレーターが結果を要約し、コミット・push まで実施する。

## 運用ルール

- 常に root/template を同期する。
- 1ターンで1つの主タスクを完了させる（必要なら小分け）。
- 完了時は以下を必ず報告する。
  - 変更ファイル
  - テスト結果
  - コミットID

## 状態管理

- 提案・選定・進行状況は docs/custom_agent_task_board.md で管理する。
- 進捗ログは docs/issue_roadmap.md にも追記する。
