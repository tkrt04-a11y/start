# Release Notes (2026-03-01, Detailed)

## Overview
This release strengthens the project’s operational foundation across automation, observability, schema governance, and CI feedback loops.

## What’s New

### 1) Pipeline Operations
- Added daily/weekly/monthly pipeline scripts.
- Added Windows task registration helpers for scheduled execution.
- Added alert webhook behaviors:
  - format switching (`generic` / `slack` / `teams`)
  - retry with exponential backoff
  - duplicate suppression via alert signature cooldown/TTL

### 2) Metrics, Health, and Reporting
- Added metrics aggregation helpers and threshold checks.
- Added operational health score and penalty/factor breakdown.
- Added ops report generation (Markdown/HTML/JSON) and report index generation.
- Added weekly artifact integrity verification and machine-readable verification output.
- Added weekly failure diagnostic generation for easier incident triage.

### 3) GitHub Integration
- Added promoted-action sync to GitHub Issues.
- Added period-aware deduplication (`action_hash + period_key`) to avoid duplicate issue creation.
- Added rate-limit aware retries for GitHub API interactions.
- Added PR comment generation for metrics checks, including runbook/retry guidance context.

### 4) Schema Governance
- Added `schema_version` support and compatibility policy.
- Added JSON schema validation CLI and CI checks.
- Enforced major-level compatibility verification in automation paths.

### 5) Dashboard Enhancements
- Added issue-sync monitoring signals.
- Added weekly failure diagnostic summary views.
- Added artifact-integrity summary display in operational dashboard paths.

## CI/Workflow Changes
- Expanded CI to validate schema and compatibility for key JSON outputs.
- Added weekly ops workflow for periodic report generation and artifact publication.
- Added failure-time diagnostic artifact upload path.

## Documentation Updates
- Updated runbook and schema compatibility documentation.
- Updated roadmap and release-facing operational guidance.
- Added/updated changelog entries for release traceability.

## Security and Risk Notes
- Per commit-scope audit, no real API keys/tokens/private keys were found in committed files.
- `.gitignore` was strengthened to avoid accidental commits of local generated artifacts and survey data.
